1.3 Código fuente, código objeto y código ejecutable



1.3.1 Tipos de código

Cuando escribimos un código, pasa por distintos estados hasta ejecutarse.


    Código Fuente  →  Código Objeto  →  Código Ejecutable
Código fuente Código escrito por el programador en lenguaje de alto nivel. No es ejecutable directamente.
Código objeto Resultado de la compilación. Representación intermedia de bajo nivel. No es ejecutable.
Código ejecutable Resultado del enlazado con librerías. Puede ejecutarse en el ordenador.

Clave: El código objeto no lo entiende ni el programador ni directamente la máquina.



1.3.2 Compilación

La compilación convierte el código fuente en código objeto.

Proceso general


    Programa Fuente
          │
          ▼
      COMPILADOR
          │
      ¿Errores?
        │       │
       Sí      No
        │       ▼
    (Corregir)  Programa Objeto
                    │
                    ▼
                 ENLAZADOR
               (+ Librerías)
                    │
                    ▼
            Programa Ejecutable

Importante: Si hay errores, no se genera código objeto.

Fases internas del compilador

Análisis léxico Divide el código en tokens.
Análisis sintáctico Comprueba que se cumplen las reglas de sintaxis.
Análisis semántico Verifica coherencia de tipos y operaciones.
Generación de código intermedio Crea representación intermedia.
Optimización Mejora el código intermedio.
Generación de código Produce el programa objeto.

Enlazador

Inserta las librerías necesarias en el código objeto para generar el programa ejecutable.

Claves de examen:

  • Compilador → genera código objeto.
  • Enlazador → genera ejecutable.
  • Orden de fases: Léxico → Sintáctico → Semántico.


1.3.3 Máquinas virtuales

Una máquina virtual es un software capaz de ejecutar programas como si fuese una máquina real.

Permite un entorno de ejecución independiente del hardware y del sistema operativo.

Clasificación


    Máquinas Virtuales
    ├── De sistema
    └── De proceso

Máquinas virtuales de sistema

  • Virtualizan un sistema operativo completo.
  • Simulan una máquina real.
  • Permiten ejecutar distintos sistemas operativos.

Máquinas virtuales de proceso

  • Se ejecutan como proceso dentro del sistema operativo.
  • Soportan un único programa.
  • Se inician al comenzar y finalizan al terminar el programa.

Ejemplo: JVM

Permite ejecutar programas Java en cualquier plataforma.

Idea clave: Independencia respecto al hardware.
